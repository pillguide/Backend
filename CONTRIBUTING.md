# Contribution Guide


## Rule

- 설명할 수 있는 코드만 커밋할 것
- AI 코드 그대로 복사하지 말것
- 반드시 커밋 전 빌드하여 로컬에서 실행되는지 먼저 확인할 것
- 이슈 발생 시 팀원에게 전파

---

## Workflows

1. 이슈 템플릿을 활용해 이슈 생성
2. `develop` 브랜치에서 새 작업 브랜치 생성
3. 기능 구현 후 의미 있는 단위로 커밋
4. `develop` 브랜치로 PR 생성
5. 리뷰를 받고, 승인 후 `develop` 브랜치에 머지

---

## Git Branches

모든 작업은 `develop` 브랜치에서 시작

### Branches Naming

**`<이슈타입>/#<이슈번호>-<작업내용 혹은 기능>`**

- **`<이슈타입>`**: 생성한 이슈의 타입
- **`<이슈번호>`**: GitHub 이슈 번호 (ex. `#12`)
- **`<작업내용>`**: 해당 브랜치의 작업내용 혹은 기능, 도메인 중 택1 

### Issue Type

| 타입 | 설명 | 
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 작업 |
| `refactor` | 코드 리팩토링 |
